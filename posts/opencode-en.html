What is OpenCode?

OpenCode is an open-source programming agent under the MIT license, designed primarily to operate in the terminal (CLI/TUI), although it also has desktop applications and extensions for various IDEs. With over 149,000 stars on GitHub, it has established itself as one of the most popular and complete tools for AI-assisted development.

How does OpenCode work?

Unlike a simple chatbot, OpenCode operates through an "agentic loop", allowing it to analyze files, execute shell commands, write code, and perform tests autonomously until a task is completed. Its workflow is divided into two fundamental states that the user can toggle with the Tab key:
  • Plan Mode: A read-only mode ideal for exploring unknown codebases, performing architectural analysis, or designing strategies without the risk of modifying files.
  • Build Mode: The default action mode, where the agent has write permissions to edit files and execute real commands on the system.

Key Technical Features

OpenCode stands out for its ability to deeply understand a project's context via the /init command, which generates an AGENTS.md file summarizing the structure and conventions of the codebase. It also integrates advanced technologies such as:
  • LSP (Language Server Protocol): To receive real-time diagnostics and compilation errors directly from the programming language used.
  • MCP (Model Context Protocol): Which allows connecting the AI with external tools like databases, web browsers, or cloud services.
  • Skills and Plugins: Packaged knowledge and extensions that modify the agent's behavior based on developer needs.

Differences with other Models

While tools like Claude Code are strictly tied to the Anthropic ecosystem, OpenCode is provider-agnostic. This means it supports over 75 model providers, allowing the user to choose between premium models like GPT-4 or Claude 3.5 Sonnet, or even local models running entirely offline via Ollama or LM Studio. Finally, OpenCode offers greater data sovereignty and cost control. Under a "local-first" philosophy, the code remains on the developer's machine and allows for leveraging existing subscriptions to services like GitHub Copilot or ChatGPT Plus, avoiding additional token payments if local configurations are used.
